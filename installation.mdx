# Installation

This guide will help you install and configure @rnmapbox/maps for iOS, Android, and Expo projects.

## Prerequisites

- React Native 0.79 or higher
- Node.js installed
- A [Mapbox access token](https://account.mapbox.com/auth/signup/)

## Step 1: Install the Package

Install @rnmapbox/maps using your preferred package manager:

<CodeGroup>

```bash npm
npm install @rnmapbox/maps
```

```bash yarn
yarn add @rnmapbox/maps
```

```bash pnpm
pnpm add @rnmapbox/maps
```

</CodeGroup>

## Step 2: Platform-Specific Setup

Choose your platform for detailed setup instructions:

<Tabs>
  <Tab title="iOS">
    ### iOS Configuration

    After installing the package, you need to configure your iOS project.

    #### Update Podfile

    Add the following hooks to your `ios/Podfile`:

    ```ruby
    pre_install do |installer|
      $RNMapboxMaps.pre_install(installer)
      # ... other pre install hooks
    end

    post_install do |installer|
      $RNMapboxMaps.post_install(installer)
      # ... other post install hooks
    end
    ```

    #### Install Pods

    Navigate to the iOS directory and install dependencies:

    ```bash
    cd ios
    pod install
    cd ..
    ```

    #### Add Location Permissions (Optional)

    If you want to show the user's location with the `UserLocation` or `LocationPuck` component, add location permissions to your `ios/Info.plist`:

    ```xml
    <key>NSLocationWhenInUseUsageDescription</key>
    <string>Show current location on map.</string>
    ```

    <Note>
    See [Mapbox iOS documentation](https://docs.mapbox.com/ios/maps/guides/user-location/#request-temporary-access-to-full-accuracy-location) for more details on location permissions.
    </Note>

    #### Custom Mapbox Version (Optional)

    To use a specific Mapbox SDK version, add this to the top of your `Podfile`:

    ```ruby
    $RNMapboxMapsVersion = '= 11.18.2'
    ```

    <Warning>
    If you set a custom version, make sure to update it when upgrading @rnmapbox/maps to avoid compatibility issues.
    </Warning>
  </Tab>

  <Tab title="Android">
    ### Android Configuration

    Configure your Android project to use Mapbox Maps SDK v11.

    #### Add Mapbox Maven Repository

    Add the Mapbox Maven repository to your `android/build.gradle` file:

    ```groovy
    allprojects {
        repositories {
            // ... other repos
            maven {
                url 'https://api.mapbox.com/downloads/v2/releases/maven'
            }
            // ... more repos
        }
    }
    ```

    <Note>
    Mapbox has removed the authentication requirement for downloads, so `MAPBOX_DOWNLOADS_TOKEN` is no longer needed.
    </Note>

    #### Custom Mapbox Version (Optional)

    To specify a custom Mapbox SDK version, add this to `android/build.gradle` in the `buildscript > ext` section:

    ```groovy
    buildscript {
        ext {
            RNMapboxMapsVersion = '11.18.2'
        }
    }
    ```

    <Warning>
    If you set a custom version, make sure to revisit it when updating @rnmapbox/maps to avoid build errors.
    </Warning>

    #### Troubleshooting: libc++_shared.so Conflicts

    If you encounter errors about duplicate `libc++_shared.so` files, add this to your `android/app/build.gradle`:

    ```gradle
    android {
        packagingOptions {
            pickFirst 'lib/x86/libc++_shared.so'
            pickFirst 'lib/x86_64/libc++_shared.so'
            pickFirst 'lib/arm64-v8a/libc++_shared.so'
            pickFirst 'lib/armeabi-v7a/libc++_shared.so'
        }
    }
    ```
  </Tab>

  <Tab title="Expo">
    ### Expo Configuration

    <Warning>
    This package **cannot** be used in the "Expo Go" app because it requires custom native code.
    </Warning>

    #### Install with Expo CLI

    ```bash
    expo install @rnmapbox/maps
    ```

    #### Add Config Plugin

    Add the config plugin to your `app.json`, `app.config.js`, or `app.config.ts`:

    ```json
    {
      "expo": {
        "plugins": [
          [
            "@rnmapbox/maps",
            {
              "RNMapboxMapsVersion": "11.18.2"
            }
          ]
        ]
      }
    }
    ```

    #### Add Location Permissions (Optional)

    To use the user's location, install and configure expo-location:

    ```bash
    npx expo install expo-location
    ```

    Add it to your plugins array:

    ```json
    {
      "expo": {
        "plugins": [
          [
            "expo-location",
            {
              "locationWhenInUsePermission": "Show current location on map."
            }
          ]
        ]
      }
    }
    ```

    #### Rebuild Your App

    After configuration, rebuild your app:

    ```bash
    npx expo prebuild
    npx expo run:ios
    # or
    npx expo run:android
    ```

    <Note>
    See the [Expo custom native code guide](https://docs.expo.io/workflow/customizing/) for more information.
    </Note>

    #### Custom Mapbox Version

    You can override the native SDK version in the plugin configuration:

    ```json
    {
      "expo": {
        "plugins": [
          [
            "@rnmapbox/maps",
            {
              "RNMapboxMapsVersion": "11.18.2"
            }
          ]
        ]
      }
    }
    ```
  </Tab>
</Tabs>

## Step 3: Verify Installation

Verify your installation by running your app:

<CodeGroup>

```bash iOS
npm run ios
# or
yarn ios
```

```bash Android
npm run android
# or
yarn android
```

</CodeGroup>

## Troubleshooting

### iOS: Pod Install Fails After Upgrade

If you see an error like `could not find compatible versions for pod "MapboxMaps"`, run:

```bash
cd ios
pod update MapboxMaps
cd ..
```

### Android: Maven Repository Issues

Ensure the Mapbox Maven repository is added **after** other repositories in your `build.gradle`.

### Expo: Module Not Found

Make sure you've run `npx expo prebuild` after adding the plugin to regenerate native code.

## Supported Versions

- **Package Version**: 10.3.0-rc.0
- **Mapbox iOS SDK**: ~> 11.18.2
- **Mapbox Android SDK**: 11.18.2
- **React Native**: 0.79+
- **Node.js**: 22.16.0 (recommended)
